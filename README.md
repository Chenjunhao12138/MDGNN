This repository implements the MD-GNN (Multi-modal Drug Graph Neural Network) model for predicting drug-drug interactions (DDI) from drug pairs. The model fuses three complementary information modalities: molecular structure (SMILES embeddings), textual descriptions (BioBERT embeddings), and heterogeneous network metapath features. It then applies structure-aware cross-attention and topology-adaptive gating to enable deep interaction and adaptive fusion of the multi-modal features. The overall architecture follows an encode-interact-fuse-predict pipeline. First, each modality is encoded to a fixed hidden dimension using dedicated encoders: TrimNetEncoder for structure, BioBERTEncoder for text, and HANEncoder for metapath features. Second, the model computes cross-drug attention outputs for each modality pair (structure-text, structure-metapath, text-metapath) using the StructureAwareAttention module, which incorporates a topological bias from the DDI graph. Third, the three attention outputs are concatenated and, together with a topological context vector, fed into a TopologyAdaptiveGating module that produces three dynamic weights to combine the modality-pair representations into a final drug-pair embedding. Finally, a two-layer classifier predicts the DDI type probabilities.

The code consists of five Python files. main.py defines the full MDGNN class and includes a small example forward pass with randomly generated data to verify the model runs correctly. trimnet_encoder.py implements TrimNetEncoder, which applies three linear layers with LeakyReLU activations followed by a multi-head self-attention layer and residual connection to enhance molecular representations. biobert_encoder.py provides a simple two-layer MLP with ReLU and dropout to map 768‑dimensional BioBERT embeddings into the hidden space. han_encoder.py implements HANEncoder, which uses a semantic attention mechanism to fuse multiple metapath features (one per metapath) into a single vector. mdgnn_modules.py contains the two core modules: StructureAwareAttention computes a scaled dot-product attention with additive graph bias, and TopologyAdaptiveGating concatenates the three modality-pair representations with the topological context and applies a softmax to obtain three gating coefficients.

The model expects pre-extracted features for each drug pair. For each drug, three tensors must be provided: structural features of shape (batch_size, s_in), text features of shape (batch_size, t_in) (typically 768), and metapath features of shape (batch_size, num_metapaths, p_in). The same three tensors are required for the second drug. Additionally, a tuple adj_info containing (adj_bias, s_ij) is required, where adj_bias has shape (batch_size, 1) and serves as an additive bias in the attention, and s_ij has shape (batch_size, hidden_dim*2+1) as the topological context for the gating module. The final output is raw logits of shape (batch_size, num_classes) that can be used with cross‑entropy loss for training or converted to probabilities by softmax for inference.

To run the example, simply execute python main.py after installing PyTorch (>=1.9.0) and a compatible Python (>=3.8). The example constructs random inputs with batch size 4 and default configuration (s_in=128, t_in=768, p_in=64, hidden_dim=64, num_classes=167), runs one forward pass through the model, and prints the output shape. For integration into a real training loop, instantiate the model with the appropriate configuration, define an optimizer and loss function, and iterate over a data loader that provides (drug_i, drug_j, adj_info, labels). The model does not include any raw data preprocessing; users must generate the input embeddings separately (e.g., from SMILES strings using a molecular graph encoder, from drug descriptions using BioBERT, and from a heterogeneous network using HAN). The TrimNetEncoder currently compresses the sequence dimension to 1 for simplicity, which reduces fine‑grained atom‑level interactions; users who need more expressive structural encoding should modify it to accept a sequence of atom representations. If the topological bias is not needed, adj_bias can be set to zero. The context dimension in the gating module is hard‑coded as hidden_dim*2+1, which must match the actual dimension of s_ij. This implementation is self‑contained and does not rely on heavy external libraries beyond PyTorch. For any questions or contributions, please refer to the original paper citation included in the code comments.
